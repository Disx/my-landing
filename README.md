# net-admin.pro — сайт (Hugo)

Лёгкий статический сайт на Hugo: одностраничная посадочная (услуги, процесс, форма) + раздел «Заметки» под SEO-контент. Деплой — GitHub Pages со своим доменом.

---

## 1. Что заполнить перед публикацией (5 минут)

Почти всё правится в **одном файле — `hugo.toml`**, блок `[params]`:

| Параметр | Что вписать | Зачем |
|---|---|---|
| `city` | **город (напр. «Минск»)** | ⚠️ без него слабое локальное SEO — заполни обязательно |
| `region` | область/страна | локальное SEO |
| `telegram` | логин без `@` (напр. `dis_uuid`) или оставь пустым | контакт |
| `showAddress` / `address` | `true` + адрес, если есть офис с приёмом | разметка адреса |
| `phoneDisplay` / `phoneHref` / `email` | уже стоят твои текущие | контакты + JSON-LD |

Ещё два места:
- **Форма:** в `layouts/index.html` поле `access_key` — вставь ключ Web3Forms (см. §4).
- **Картинка превью:** положи `static/img/og.png` (1200×630) — как сайт выглядит при репосте в мессенджерах/соцсетях.

Глобальная замена `Office-Admin` на другое название бренда — тоже в `hugo.toml` (`businessName`).

---

## 2. Локальный запуск

```bash
hugo server -D
# открой http://localhost:1313
```

## 3. Деплой

Структуру из архива кладёшь в корень репозитория (заменяя старую). В репозитории `.github/workflows/deploy.yml` уже настроен: при `git push` в `main` сайт сам собирается и публикуется на GitHub Pages.

Один раз в настройках репозитория: **Settings → Pages → Build and deployment → Source = GitHub Actions**.

```bash
git add -A
git commit -m "Новый шаблон сайта"
git push
```

Файл `static/CNAME` уже содержит `net-admin.pro` — домен подхватится автоматически.

## 4. Форма обратной связи

### Сейчас — Web3Forms (проще всего, без сервера)
1. Зайди на web3forms.com, введи свой email (`disx@yandex.ru`), получи **Access Key**.
2. Вставь ключ в `layouts/index.html`:
   ```html
   <input type="hidden" name="access_key" value="СЮДА_КЛЮЧ">
   ```
3. Всё — заявки падают на почту. Бесплатно, без аккаунта на сервере.

### Позже — свой n8n → Telegram (когда захочешь)
В `static/js/main.js` поменяй одну строку:
```js
var ENDPOINT = "https://твой-домен/webhook/zayavka"; // URL вебхука n8n
```
n8n принимает обычный JSON (`name`, `organization`, `phone`, `message`) → дальше нода Telegram. Нужен публичный HTTPS-вебхук на Debian-боксе (Cloudflare Tunnel — самый простой путь) и CORS-заголовок `Access-Control-Allow-Origin: https://net-admin.pro` в ответе вебхука.

## 5. SEO вне сайта — без этого топа не будет

Вёрстка и разметка уже сделаны (semantic HTML, мета, Open Graph, JSON-LD `ProfessionalService`, `sitemap.xml`, `robots.txt`). Но для локальных IT-услуг **главное — не на сайте**. По убыванию отдачи:

1. **Google Business Profile** (business.google.com) и **Яндекс.Бизнес** (business.yandex.ru) — карточки компании с категорией «IT-услуги/системное администрирование», городом, телефоном и сайтом. Для локальной выдачи это даёт больше, чем что-либо на самом сайте.
2. **Подтверди сайт и отдай sitemap:**
   - Google Search Console → добавь `net-admin.pro`, отправь `https://net-admin.pro/sitemap.xml`.
   - Яндекс.Вебмастер → то же самое.
3. **Единый NAP** — название/телефон/город одинаково на сайте, в Google и Яндексе. Расхождения путают поисковик.
4. **Контент = ранжирование.** Раздел «Заметки» под это и сделан: 1–2 короткие статьи в месяц по реальным задачам («настройка VPN для офиса», «защита от шифровальщиков», «миграция 1С»). Это твой контент-бренд, переложенный в текст под поиск. Пустой раздел не вредит, но и не помогает — работает только наполнение.
5. **Отзывы** в карточках Google/Яндекс — просить у довольных клиентов. Сильный локальный фактор.

## Структура

```
hugo.toml                 конфиг + все контакты/гео (единый источник правды)
content/
  _index.md               мета главной
  blog/                    «Заметки» (markdown-статьи)
data/services.yaml         список услуг (правится без вёрстки)
layouts/                   шаблоны (дизайн)
static/                    css, js, robots.txt, favicon, CNAME, img/
.github/workflows/         автодеплой
```
Чтобы добавить услугу — допиши блок в `data/services.yaml`. Чтобы добавить статью — положи `.md` в `content/blog/`. Вёрстку трогать не нужно.
